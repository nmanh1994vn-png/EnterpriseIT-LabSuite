# SOC Windows Event Detection Lab

## Lab Overview

This lab simulates a basic Security Operations Center (SOC) monitoring environment to demonstrate how security analysts detect suspicious activity using endpoint telemetry and a SIEM platform.

The objective of the lab is to generate attack activity and detect it using **Sysmon logs and Splunk Enterprise SIEM**.

Simulated attacker behavior includes **network scanning and authentication failures**, allowing security events to be captured and analyzed in Splunk.

---

## Lab Environment

**Attacker Machine**

* Kali Linux

**Target Machine**

* Windows 11 Enterprise

**Security Monitoring Tools**

* Sysmon
* Splunk Enterprise
* Splunk Universal Forwarder

---

## Tools Used

| Tool                       | Purpose                                                   |
| -------------------------- | --------------------------------------------------------- |
| Sysmon                     | Generates detailed Windows system telemetry               |
| Splunk Enterprise          | Security Information and Event Management (SIEM) platform |
| Splunk Universal Forwarder | Sends endpoint logs to Splunk                             |
| Nmap                       | Network reconnaissance and scanning                       |
| Windows Event Logs         | Authentication and system activity monitoring             |

## Attack Simulation

To generate security telemetry, the following attacker activities were simulated:

* Network reconnaissance using **Nmap TCP SYN scan**
* Failed authentication attempts against the Windows endpoint
* Monitoring endpoint telemetry generated by **Sysmon**

These actions simulate common attacker behavior during the reconnaissance and initial access phases of an attack.

---

## Detection Queries Used

### Detect Failed Login Attempts

```id="f1query"
index=main EventCode=4625
```

Event ID **4625** represents failed authentication attempts.
Multiple occurrences may indicate **brute-force login attempts**.

---

### Detect Sysmon Network Activity

```id="f2query"
index=main EventCode=3
```

Sysmon **EventCode 3** records network connections initiated by processes on the endpoint.

This allows analysts to detect:

* Network scans
* Suspicious outbound connections
* Malware command-and-control traffic

## Conclusion

This lab demonstrates how a SOC analyst can monitor endpoint telemetry, analyze SIEM logs, and detect suspicious activity in a controlled lab environment.

Using **Sysmon and Splunk**, it is possible to identify attacker reconnaissance activity and authentication anomalies, providing valuable visibility into potential security incidents.
